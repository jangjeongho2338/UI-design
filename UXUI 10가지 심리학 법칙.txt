UX/UI 10가지 심리학 법칙

1. 제이콥의 법칙

	- 제이콥이 스스로 인간 본성의 법칙이라고 한 법칙
	- 사용자가 다른 웹사이트를 통한 경험을 바탕으로 디자인관례에 대한 기대치를 형성하는 경향을 보임
	- 디자이너들이 일반적인 디자인 관습을 따를 것을 권장
	- 특이한 인터페이스 사용시 사용자의 불만, 혼란 야기 가능 > 서비스 이탈률 증가로 이어짐
	- 멘탈모델과도 관련 있음 (우리가 어떤 시스템에 관해 특히 그 시스템의 작동 방식에 관해 알고 있다고 생각하는 것)

제이콥의 법칙 활용 팁
	- 익숙한 UI 패턴 활용
	- 로그인 방식, 버튼 위치, 인터페이스는 사용자가 아는 구조를 그대로 사용
	- 차별화는 기능이 아닌 감성부터
	- 구조나 흐름은 익숙하게 하고, 컬러나 애니메이션, 마이크로 인터랙션으로 감성적인 차별화 시도
	- 새로운 방식은 힌트를 함께 (슬라이드를 유도하는 화살표, 클릭가능한 UI만 표시등)
	- 좋은 UX는 신기보단 편하다에서 시작됨

2. 힉의 법칙

	- 선택 가능한 옵션 수가 증가할수록 의사 결정 시간이 기하급수적으로 증가한다는 개념 기반
	- 심리학 및 인간-컴퓨터의 상호작용 분야에서 중요한 원칙
	- RT=a+b⋅log2​(n+1) 공식
	- RT(반응시간) = 사용자가 결정하는 시간
	- n = 선택 가능한 옵션의 개수
	- a, b = 경험적 상수(문맥과 시스템이 따라 다름) (ex : a는 준비하고 반응하는 시간 b는 선택지 증가에 따른 시간 추가)
	- '정보 이론'의 개념과도 연관됨, 인간이 정보를 처리하는 과정에서 발생하는 인지적 부담을 수량화하는 데에 사용
	- UX/UI디자인에서 선택 과부화 문제를 해결하는 중요한 이론적 근거가 됨

힉의 법칙과 UX디자인의 상관관계

	- 힉의 법칙은 UX/UI디자인에서 인지적 부담을 줄이는 데 매우 중요한 역할을 함
	- 사용자는 많은 선택지가 주여질 경우 결정 피로를 경험하며 이는 서비스 이탈률 증가로 이어짐

힉의 법칙이 적용되는 주요 영역

	- 네비게이션 설계 : 직관적인 정보 계층 구조를 제공하여 사용자의 탐색 부담 줄임
	- CTA(Call-To-Action) 버튼 설계 : 핵심적인 행동을 유도해 사용자의 결정을 단순화함
	- 폼 디자인 : 불필요한 입력 필드를 최소화해 사용자 경험 개선
	- 이커머스 및 제품 필터링 : 필터 옵션을 최적화해 사용자의 검색 시간을 단축

인지 부하 이론과의 연계

	- 인지 부하 이론 : 인간의 작업 기억은 한정된 용량을 가지므로 과도한 정보가 주어질 경우 정보 처리 능력이 저하됨
	- 따라서 UX 설계는 사용자가 한번에 처리해야할 정보량을 적절히 조절해야함

힉의 법칙 활용 팁

	- 선택지 최소화 : 한번에 고려할 선택지를 줄이는 정보 처리 속도 업 ( ex : 관련 상품 그룹화 필터링 제공으로 줄임)

	- 선택지 계층화 : 메뉴 구조와 네비게이션을 논리적으로 계층화 하면 사용자의 탐색 경험을 최적화 가능
		- ex : 대시보드에 주요 메뉴를 상위 배치 후 세부 옵션은 드롭다운으로 제공

	-  점진적 정보 제공 : 사용자가 처음부터 모든 정보를 받는다면 인지적 과부화 발생 가능성 큼 
		- 따라서 필요한 정보만 제공 후 나머지는 점진적으로 노출하는 것이 효과적임 
		- ex) 회원가입시 기본 정보만 처음에 입력받고 다음 단계에서 하나씩 받는 방식을 쓰면 이탈률이 줄 수 있음

	- 시각적 계층 구조 활용 : 가장 중요한 선택지를 색깔로 강조하고 버튼 크기등 UI요소의 가독성을 개선해야함

힉의 법칙을 적용한 UX디자인 사례

	- Apple의 제품선택 UI : 사용자에게 기본적으로 두 가지 모델 중 선택하게 하고 추가적인 사양 설정 유도
	- Netflix의 콘텐츠 추천 알고리즘 : 사용자가 너무 많은 콘텐츠를 탐색하지 않도록 개인화된 추천 시스템을 적용
	- Amazon의 원클릭 구매 시스템 : 최소 단계만 거쳐 결제가 완료되도록 설계함
	- 구글의 검색 자동 완성 : 검색어를 입력할 떄 자동 완성 기능을 제공해 선택지를 제한함으로 검색 과정을 단순화함

모바일 UX와 힉의 법칙 적용

	- 모바일 UI에서 선택지 최소화 : 작은 화면 크기로 인해 선택지 최소화가 중요, 대표적인 방법으로 탭 구조, 아코디언 메뉴등이 있음
	- 반응 속도 고려한 UX 최적화 : 인터페이스끼리의 반응 속도가 중요함, 불필요한 애니메이션을 줄이고 가벼운 인터페이스 유지

결론적인 힉의 법칙

	- 힉의 법칙은 사용자의 선택 과정을 최적화 하고 불필요한 인지 부담을 줄이는 핵심적인 원칙
	- 이름 올바르게 적용하면 UX 만족도를 높이고 전환률 개선, 이탈률 감소, 사용자 만족도 증가등의 실질적인 효과를 기대 가능

핵심 활용 방법

	- 선택지 최소화 : 직관적이고 간결한 디자인 적용
	- 점진적 정보 제공 : 한번에 과도한 정보 제공을 하지 않음
	- 시각적 계층 구조 활용 : 핵심 요소를 강조해 사용자의 선택 유도
	- 모바일 UX 최적화 : 터치 기반 환경에 맞게 불필요한 요소 제거

3. 피츠의 법칙

	- 인간의 운동 행동을 예측하는 수학적 모델, 목표까지 거리와 목표의 크기가 작업 속도에 영향을 미친다는 개념을 기반함
	- 인간-컴퓨터 상호작용 및 UX/UI 디자인에서 버튼,아이콘,터치 타겟 설계의 핵심 원칙으로 활용
	- T=a+b⋅log2​(1+D/W​) 공식
	- T : 목표에 도달하는데 걸리는 시간
	- D : 현재 위치에서 목표까지의 거리
	- W : 목표의 크기(너비)
	- a,b = 경험적 상수(환경, 조건마다 다름)

피츠의 법칙 UX/UI디자인 적용

- 피츠의 법칙을 적용하면 버튼, 링크, 터치 영역을 적절한 크기로 설계하여 사용자의 클릭 정확도를 높임

	- 버튼 크기 조정 : 버튼이 작으면 클릭하기 어려우므로 최소 모바일 기준 40~48px이상의 크기를 유지
	- 버튼 간격 최적화 : 버튼 간 거리가 너무 멀면 이동 시간이 증가하고, 나무 가까우면 실수로 클릭할수 있음
	- 중요한 버튼을 화면 가까이 배치 : 사용자가 자주 클릭하는 요소는 엄지손가락이 쉽게 닿는 영역에 배치
	- 데스크톱과 모바일에서 환경 차이 고려 : 데스크톱은 마우스 모바일은 손가락이므로 디자인 원칙이 다르게 적용되야댐

- 네비게이션 및 메뉴 디자인

	- 메뉴 크기 및 배치 최적화 : 주요 메뉴를 크고 쉽게 접근 가능한 위치에 배치	
	- 내비게이션 요소의 시각적 강조 : 중요한 메뉴나 버튼을 눈에 띄게 하면 쉽게 접근 가능
	- 드롭다운 메뉴 사용 시 거리 고려 : 사용자가 메뉴를 열고 하위 메뉴로 이동하는 거리를 줄여야함
	- 햄버거 메뉴 대신 하단 네비게이션 바 사용 : 모바일은 네비게이션 바가 접근성 좋음

- 터치 인터페이스 최적화

	- 터치 타겟 크기 증가 : 손가락 크기를 고려해 버튼과 링크의 터치 영역을 설정
	- 제스처 기반 인터랙선 적용 : 화면 스와이프, 핀치 줌 등은 거리 개념을 활용하여 직관적인 사용자 경험제공 가능
	- 폼입력 필드 크기 조정 : 입력 필드가 작으면 터치 오작동이 발생할 수 있으니 적절한 크기로 조정해야함
	- 접근성강화 : 몸이 불편한 사용자도 쉽게 인터페이스를 활용할 수 있도록 키보드 네비게이션과 음성 입력등 지원해야함

- 피츠의 법칙을 활용한 UX 디자인 사례

	- Apple ios 버튼 : 모바일 버튼 크기를 최소 44px로 설정해 터지 정확도를 높임
	- Google Material Design 버튼 크기 규정 : 기본 버튼 크기를 48px이상으로 유지하고, 주요 액션 버튼은 더 크게 디자인 하도록 권장하고 있음
	- Fitts' Law를 고려한 e-Commerce  UX 개선 : 아마전은 결제 버튼을 강조되게 배치하고 '1-Click구매'로 사용자의 클릭 부담을 최소화하는 방식을 사용함
	- 소셜 미디어 플랫폼의 UX 최적화 : 페이스북, 인스타그램, 트위터등 좋댓구 버튼을 엄지가 쉽게 닿는 곳에 배치해 편의성 패치함

- 피츠의 법칙과 웹사이트 최적화 전략

	- CTA 버튼 크게 만들기 : 구매, 회원가입등 주요버튼을 크고 눈에 잘띄게함
	- 마우스 이동 거리 줄이기 : 사용자가 원하는 액션을 수행하는데 불필요한 마우스 이동거리를 줄이면 UX가 개선됨
	- 반응형 디자인 적용 : 다양한 디바이스에서도 버튼 크기와 인터랙션 거리가 적절히 유지되야됨
	- 다크 모드 및 가독성 고려 : 피츠의 법칙을 적용할 때 가독성 좋은 색상과 디장인을 선택해야됨

- 결론 

- 피츠의 법칙은 디자인의 핵심 원칙중 하나로, 인터랙션 효율성을 최적화하는 데 필수 개념이다
- 올바르게 적용하면 사용자 경험을 개선하고 전환율을 높이는데 큰 기여를 할 수 있음

- 핵심 전략

	- 버튼 및 인터랙션 요소 크기 최적화 : 클릭 가능한 요소는 충분한 크기를 유지해야함
	- 이동 거리 최소화 : 네비게이션과 액션 버튼은 사용자 손이 닿기 쉬운 곳에 배치해야함
	- 터치 스크린 UX 개선 : 터치 타겟 크기를 키우고, 실수로 클릭할 확률 줄이기 위해 간격 조절
	- CTA 버튼 강조 : 중요한 액션 버튼은 크고 눈에 띄게 디자인함

- 피츠의 법칙을 전략적으로 활용하면 더 빠르고 직관적인 UX를 제공할 수 있으며, 사용자 만족도를 극대화할 수 있다

4. 밀러의 법칙

	- 같은 숫자 묶음이라도 통으로 나열하는 것보다 몇 덩어리로 나눠서 기억하는 것이 더 쉽다

- 7의 법칙

	- 자극에 포함되는 정보의 양에 큰 차이를 줘도 청소년의 기억 범위는 대략 7 정도로 제한된다
	- 기억 범위에 더 큰 영향을 미치는건 정보의 기본 단위인 비트의 양이 아닌 정보 덩어리의 개수
	- 하지만 실험 결과에 따르면 한 번에 기억 가능한 정보는 사람의 배경지식에 따라 7개보다 적을 수도 있음
	- 핵심은 작업 기억에 한 번에 저장 가능한 정보의 개수를 밝히는 것이 아닌 정보들을 유의미한 덩어리로 정리하면 7(+-2)개를 넘어 일정 한계가 존재하는 단기 기억을 최적화 할 수 있다는 점을 시사한다
	
- 여러 개의 정보를 유의미한 덩어리로 묶기

	- 사용성 높은 웹사이트 디자인을 위해서는 먼저 콘텐츠를 사용자들이 이해하기 쉬운 형태로 배치해야함
	- 이를 위해 자연스럽게 게슈탈트 원칙 중 하나인 근접성의 원칙을 함께 고려하게 된다
	- 근접성의 원칙 : 형태를 지각할 때 거리가 가까운 요소들을 하나로 묶어 지각하려는 경향을 말함
	- 사용자는 서로 다른 형태의 디자인 요소들이 단지 가까운 거리에 위치한다는 것만으로 이들을 하나의 덩어리나 맥락으로 분류해 지각함
	- ex) 뉴스 사이트는 최신 뉴스 헤드라인을 중요도에 따라 크기를 나뉘 정리되어 있고 그 사이 여백과 선을 둬서 사용자의 흥미있는 콘텐츠 탐색을 돕는다
	- ex) 쇼핑 사이트에서 선이나 테두리로 묶지 않고 가까이 이미지, 제품명, 가격을 둔 것 만으로도 하나의 덩어리를 형성함
	- ex) 전체 메뉴를 보면 사이트 링크는 7개를 훨씬 넘지만 각 카테고리가 명확하고 근접성의 원칙을 이용해 여백으로 분리 시켰고 결과적으로 쉽게 볼 수 있다  

- 결과 고려할점

	- 마법의 숫자 7을 내세워 불필요한 디자인 제약을 정당화 하지 말 것
	- 사용자가 쉽게 처리, 이해, 기억을 할 수 있도록 콘텐츠를 덩어리로 나눠서 정리할 것
	- 단기 기억 용량은 사람에 따라, 그리고 기존 지식과 상황적 맥락에 따라 달라진다는 것을 기억해야 함

5. 포스텔의 법칙

	- 자신이 행하는 일은 엄격하게 남의 것을 받아들일 때에는 너그럽게 라는 원칙
	- 사용자가 어떤 기기이든 기능, 입력 메커니즘, 보조 기술, 연결 속도에 관계없이 인터페이스가 계속 작동 해야함
	- 데이터를 전송하는 프로그램은 해당 데이터를 엄격하게 검사하고 데이터를 받는 프로그램은 가능한 너그럽게 받아들여야 한다는 원칙
	- 이를 통해 통신망 전체의 유연성과 확장성이 상향됨
	
- 디자인과 포스텔 법칙

	- 사용자에게 보다 쉽고 편리한 인터페이스를 제공 할 수 있다
	- 다자인은 사용자의 다양한 기술적 배경과 이해도를 수용하고 다양한 사용자의 요구를 만족 가능

- 적용 방식

	- 폼디자인 : 사용자가 폼을 채울 때 요구정보를 가능한 제한적으로 요청하는 것이 중요, 폼을 채우는데 시간과 노력을 줄여야하고 다양한 형식을 수용해야함
		- ex) 20010101을 입력하면 자동으로 2001-01-01로 인풋 텍스트가 전환되는 UX
	- 사용자 인증 : 간편하게 인증을 최소화한다
		- ex) 애플은 FaceID를 통해 아이디와 비밀번호를 입력하는 번거로움 없이 간편하게 잠금해제 가능하다, 또한 사용자의 외관변화등 요소에 너그럽게 반응
	- 오류 처리 : 사용자가 실수를 하면 가능한 그 실수를 용인하고 대신 적절한 피드백을 제공하여 사용자의 실수 수정을 도와야한다
		- ex) 전화번호 폼에 영어를 입력시 올바른 형식의 전화번호를 입력하지 않을시 다음으로 안넘어간다
	-반응형 웹디자인 : 크기에 따라 컨텐츠가 자동으로 조정되는 방식 사용자는 특정 장치나 화면 크기에 구애받지 않고 웹사이트 사용 가능
	- 서치 바 : 컨텐츠에 초점을 맞추고 스타일과 인터랙션을 점진적으로 쌓아가는 웹디자인 전략 핵심 콘텐츠를 지키며 향상된 기능을 추가하여 접근성 저해 X
		- ex) 네이버 검색 화면은 누구나 검색어를 입력할 수 있도록 음성인식을 지원함 -> 핵심기능 손상없이 검색 방법 확장됨

- 결론

	- 포스텔 법칙은 사용자의 다양한 요구와 기술적 배경을 수용하고 사용자에게 편리하고 친숙한 경험을 제공하는데 도움이 된다
	-  이 원칙은 사용자 중심의 디자인을 실현하느데 큰 도움을 줘서, 디자이너와 개발자가 유연하고 안정적인 인터페이스를 제공할 수 있게 한다

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------

6. 피크엔트 법칙

	- 피크엔트 법칙 : 가장 강렬한 순간과 마지막 순간을 가장 잘 기억하고 그것을 기준으로 평가하는 인간의 심리를 이용한 법칙
	- 전체 경험을 기억하고 평가할 때 사용자가 가장 높은 반응을 보였던 '피크'와 마지막 순간 '엔드'가 결정적인 역할을 한다는 것이다
	- 이 원칙은 사용자 경험이 어떻게 구성되고 평가되는지에 대한 통찰력을 제공함
		- ex) 우리가 제품이나 서비스를 사용하며 감정은 순간마다 변함 그러나 모든 감정이 기억되지는 않고 가장 강렬한 감정을 불려일으킨 순간과 마지막 순간의 감점이 가장 강력하게 기억되며 이 두 순간이 우리의 전체 경험을 어떻게 기억하고 평가하게 될지 결정한다

- 인지 편향

	- 인지 편향을 이해하는 것은 피크엔트 법칙을 이해하는데 도움이 된다. 인지 편향은 판단에 영향을 주는 사고나 이성의 계통적오류를 의미한다
	- 일상적인 상황에서 모든 상황에 깊이 생각하기에는 시간이 부족함 이로인해
	- 평소에는 무의식적으로 자동 반응을 통해 빠르게 일을 처리하고 복잡한 정식 노동은 필요할 때만 하는 경향이 있음 
	- 그 결과 논리적이지 않은 방향(감성, 경험, 습관, 믿음 등)으로 치우치는 계통적 오류가 발생할 수 있음
		- ex) 시험 망치고 단순해 운이 나빴다고 생각 후 스스로의 공부 방법을 돌아보지 않음

- 인지 편향의 종류

	- 확증 편향 : 우리는 기존 신념을 지키기 위해 이를 지지하는 정보에 주목하고, 이에 반하는 정보는 무가치하다고 간주하는 경향이 있다
		- ex) 커피가 몸에 나쁘다라고 믿음 > 커피가 몸에 나쁘다는 정보에는 주목 > 하루 한잔의 커피는 몸에 좋다는 연구 결과 무시
	- 기억 편향 : 기억 회상을 방해하는 편향 우리는 자신의 감정을 강하게 자극한 사건을 더 잘 기억함 이는 우리 인지 방식에도 영향을 미침
		- ex) 영화 전체가 지루했는데 마지막이 감동적이라면 > '그 영화 진짜 좋았어'라고 말함
		- ex) 반대로 여행을 즐겁게 하다 마지막날만 비가 하루종일와서 힘들었다면 > '그 여행 비와서 별로였어'라고 함
	- 최신 효과 : 최신에 입수된 정보일수록 더 잘 기억하는 경향이 있다는 것을 의미
		- ex) 면접에서 초반에 실수해도 마지막을 잘 마무리하면 마지막 인상을 더 좋게 기억함
		- ex) 시험 공부 할 때 마지막에 본 내용이 더 잘 떠오르는 것도 같은 원리임

- 피그엔드사례

	- 피크 상황 : 메일 발송 직전의 상황에 귀여운 이미지를 넣어 불안한 순간을 유쾌하고 긍정적인 감정으로 바꿈
	- 엔드 상황 : 메일 발송 후 메시지로 성공적을 보냈다는 듯이 출력하면 사용자에게 잘 마무리 됬다는 안정감과 만족감을 줌
	- 피크 상황 : 음식 배달 앱에서 주문후 사용자의 부정적 감정이 생시는걸 막기위해 조리 중이라는 단순한 말대신 일러스트, 실시간 위치. 도착 시간등으로 지루함을 기대감으로 바뀌게 함
	- 엔드 상황 : 사용자가 잘못된 주소를 입력해 404오류를 보면 부정적 감정이 들지만 404페이지에 귀여운 일러, 재치있는 문구, 홈으로 버튼등을 통해 오류는 났지만 기분 나쁘지 않는 경험이라고 인식 시켜 긍정적인 기억으로 바꿈

- 결론

	- 우리의 인지편향은 경험을 평가하는데 큰 영향을 미친다
	- 특히 피크엔드 법칙은 강렬한 순간과 마지막 순간을 중심으로 경험을 평가한다는 것을 보여준다
	- 따라서 우리는 사용자가 전체 경험을 긍정적으로 평가하고 기억할 수 있도록 중요한 순간에 주의를 기울여야함
	- 이를 통해 사용자와의 긍정적인 관계형성와 서비스의 가치를 높일 수 있다 (사용자에게 최상의 경험을 제공 할 수 있다)

7. 심미적 사용성 효과

	- 사용자는 미적으로 뛰어난 디자인을 실제 그것이 올바른지에 관계없이 미적으로 뒤떨어지는 본래의 것보다 사용하기 좋다고 생각하는 현상을 말한다
	- 만약 같은 기능의 두제품이 있을 때 윈도우(Window)보다 맥(mac)을 구매하는 사람들의 심리
		> Mac의 디자인이 더 멋있다고 생각하는 것이 하나의 이유 일수도 있다
	- 오히려 '실제 기능성과 관계가 없다'보다 디자인이 좋다 보니 불편함에 너그러워 지고 더 적극적이라면 그 자체의 장점을 찾으려고도 함
	- 실제로 mac과 windows가 성능이 똑같아도 가격이 싼 windows 대신 가격이 비싼 mac을 쓰는 이유에 디자인의 요소도 크다

- 심미적 사용성에 의한 효과

	- 취급이 간단할 것 : 서포트나 트레이닝이 필요없으니 구매
	- 긍정적 평가 및 인상 : 세일즈 증가
	- 실패를 견뎌 보려 한다 : 문의 감소
	- 친구에게 공유 : 세일즈 증가

- 웹에서의 응용

	- 아무리 사이트가 훌륭한 콘텐츠, 정보가 있어도 외형이 나쁘다면 방문자가 들어갔다 내용을 보지 않고 이탈할 수 있다
	- 휼륭한 디자인의 e-commerce 사이트라면 상품 가격이 다소 비싸도 방문한 사람이 구매할 수 있다

8. 폰레스토프 효과

	- 비슷한 사물이 여러 개 있다면 그 중에서 가장 차이 나는 한 가지만 기억할 가능성이 크다

- 개요

	- 인간은 진화를 통해 시각과 인지 처리 체계가 매우 정교하게 발달됨 그래서 기억보다 인식(지각)이 우선되고 시각전 자극에 더 즉각적으로 반응함
	- 우리가 무엇에 관심을 두고 집중하는가는 단순한 의지가 아니라, 본능적 시각 반응과 목표 달성 욕구의 영향을 함께 받음
	- 한마디로 시각적 강조를 사용해 사용자의 주의를 끌고 목표로 안내 가능하다는 뜻임
	- 하지만 여러 요소가 서로 경쟁한다고 느껴질 정도로 너무 많이 강조하면 사용자가 필요로 하는 정보를 찾기 어려워짐

- 기원

	- 독일의 여성 심리학자이자 소아과 의사였던 헤드윅 존 레스토프 이름에서 유래
	- 격리효과를 이용한 연구에서 참가자에게 유사한 항목으로 구성한 목록을 보여주고 사람들이 그 중 뚜렷하게 구분되는 항목을 가장 잘 기억한다고 밝힘	- 인간은 다른 요소와 시각적으로나 개념적으로 분리된 항목을 더 잘 기억한다는 뜻

- 특징

	- 선택적 주의력 : 주변 사물에 집중하는 능력은 용량과 지속 시간에 제한이 있으므로 우리는 관련없는 정보를 희생하고 관련 있는 정보에 집중함
		- 이러한 현상은 인지심리학에서 선택적 주의력이라고 일컫는 생존본능임
		- 인간이 평소에 주변 세상을 인지할 때 뿐아니라 생사를 넘나드는 중요한 순간에 감각정보를 처리할 때도 중요함
	- 배너 무시 : 사용자가 광고라고 인식하는 요소를 무시하는 경향을 가리키는 용어임
		- 꼭 필요한 콘텐츠 요소라 해도 조금이라도 광고처럼 보인다거나 광고 근처에 있으면 무시당할 수 있음
	- 변화 무시 : 시각적인 단서가 충분하지 않거나 주의력이 다른 곳에 분산되어 있을 때 사람들이 변화를 잘 알아채지 못하는 경향
		- 인간의 주의력은 한정된 자원이라 임무를 효율적으로 완수하는 것과 무관하다고 여겨지는 정보를 대체로 무시한다
	- 인간의 주의력은 제한되어 필요한 정보만 선택적으로 받아드리며, 이로 인해 광고 요소를 자동으로 무시하거나, 주의가 분산되면 변화도 인식하지 못하는 현상이 나타날 수 있음

- 사례

	- Google의 플로팅 액선 버튼 : 화면에서 가장 기본적인 동작을 수행함
		- 구글은 이 요소의 디자인등에 대한 가이드라인을 제시함으로서 다양한 제품과 서비스에서 일관성을 유지 (시각적 일관성)
		- 그 결과 사용자가 알아보는 익숙한 패턴이 되어 사용자의 경험을 안내하는 역할을 함 (예측 가능한 인터랙션)
	- 드롭박스 : 우리가 사용하는 서비스의 대부분은 요금제를 운영하며, 그중 추천 요금제를 강조하는 경우가 많음
		- 강조하는 항목을 차별화하기 위해 시각적 단서를 더하는 방법을 많이 씀 (버튼의 색상, 형태를 바꿔 강조)
	- 핸드폰의 알람 : 알람은 사용자에게 할 일이 생겼다는 사실을 통보함  ex) 앱의 빨간색 숫자
		- 이 요소는 앱이나 서비스에 쓰이고 좋던 싫던 사용자의 주의를 환기시킴 (눈에 확 띄임)
	- 뉴스 웹사이트 : 통상적으로 뉴스 사이트에는 많은 헤드라인과 이미지, 광고보다 도드라지게 강조하는 추천 콘텐츠가 있음
		- 추천 콘텐츠의 크기를 키워서 다른 콘텐츠와 대조를 이루게 하는 패턴이 일반적으로 사용됨 (시각적 대비를 통한 강조)
		- 독자는 이렇게 강조된 콘텐츠 칼럼에 실린 이 모든 사례가 보여주듯이 시각적 대비를 이루는 방법은 다양함

- 중요 사항

	- 시각적 차별화를 적용할 시점과 빈도의 문제(절제) : 폰 레스토프 효과는 남용하지 말고 명확한 목적이 있을 경우 사용
		- 아무 것도 강조하지 않은 것보다 안좋은 것이 너무 많은 요소를 강조하는 것임
		- 너무 강조요소가 많으면 강조요소나 콘텐츠가 도드라지지 않을 뿐아니라 사용자에게 시각적 부담감을 안김
	- 자신이 강조하려는 시각전 속성이 다양한 사용자에게 어떤 영향을 미치는지 파악 필요 (접근성) : 예시로 색맹이 있는 사용자는 특정한 색상을 구분하지 못함 
		- 색상이라는 속성으로만 강조하려면 이러한 사용자에게는 훌륭한 경험을 제공 못함
		- 이러한 사용자가 불편하지 않도록 전경 요소와 배경 요소의 색상이 충분한 대비를 이루기 해야함

- 정리 

	- 폰 레스토프 효과는 대조를 활용해서 사용자가 가장 중요한 콘텐츠에 주의를 기울이게 하는 방법을 알려줌
	- 이 효과를 잘 사용하면 종요한 동작이나 정보를 강조할 방법을 알고 사용자가 목표 달성에 필요한 요소를 빠르게 알아보도록 디자인 가능
	- 시각적 : 시각적이 차별화 요소는 사용자의 주의를 이끔
		- 중요한 정보나 핵심 동작은 시각적으로 눈에 띄게 해라, 단 시각적 요소를 강조할 때는 강조한 수에 제한을 두어서 일부만 강조
		- 그렇게 각 요소간의 경쟁을 피하고 가장 중요한 항목이 광고로 오인되지 않게 하라
	- 디자인 : 디자인 강조에 사용되는 시각적 속성이 색맹등의 일부 사용자에게 어떻게 인식될지, 움직임에 민감한 사용자에세는 어떤 영향을 미칠지 고려해야함
		- 특정 요소를 강조할 때 색상에만 의존하면 색맹이나 저시력인 사용자가 배제될 수 있다는 사실을 기억해라
		- 움직임을 활용해 대비를 전달 할 때는 움직임에 민감한 사용자를 주의 깊게 고려해라

9. 테슬러의 법칙

	- 복잡성 보존의 법칙이라고도하며 모든 시스템에는 더 줄일 수 있는 일정 수준의 복잡성이 존재한다는 법칙
	
- 개요

	- 애플리케이션이나 프로세스의 복잡성은 사용자, 디자이너, 개발자 중 누가 부담해야 하는가(누군가는 그 복잡성을 감당해야한다는 사실)
	- 좋은 UX는 사용자의 복잡성을 최소하고 디자이너, 개발자가 느끼는 복잡성을 대신 처리함, 하지만 나쁜 UX는 사용자에게 복잡성을 떠넘김

- 기원

	- 1980년대 컴퓨터 과한자 래리 테슬러가 인터랙션 디자인 언어를 개발 업무 수행하던 시절에 탄생
	- 인터랙션 디자인 언어 : 테스크탑 및 테스크탑 퍼블리싱 개발의 핵심인 인터랙션 시스템의 구조와 기능을 정의하는 원칙 표준 모범 사례 모음집을 가리킴

	- 사용자가 애플리케이션과 인터렉션하는 방식이 애플리케이션만큼 중요함
		-> 양쪽의 복잡성을 줄이는 것이 중요해짐
		-> 이렇게 남은 복잡성은 개발 과정이나 사용자 인터랙션 단계 중 어느 쪽이든 처리해야함 

- 사례
	
	- 이메일 : 이메일을 보낼때 받는 사람의 이메일을 다 치는 번거로움(복잡성)을 줄이기 위해서
		- 보낸 사람정보는 로그인한 사용자 이메일로 채우고
		- 받는 사람정보는 입력하고 있는 주소를 기반으로 추천 이메일을 제안함
		- 이 2가지 방식으로 복잡성이 사라지진 않지만 사용자가 해야하는 수고를 줄인다
		- 지메일은 더 나아가 사용자의 입력 내용을 스캔후 맥락에 맞는 단어나 구절을 제안해서 문장을 쉽게 마무리하도록 돕고 타이핑 시간을 절약해줌
	- 결제 프로세스 : 예전에는 온라인에서 물건을 구매서 결제나 배송관련 정보를 반복적으로 제공해야함
		- 요즘 온라인 결제는 앱 안에 배송지 정보와 결제 카드를 저장하고 한번의 클릭으로 결제 카드와 배송지 정보를 입력할 수 있음

-  정리
	- 모든 프로세스는 디자인시 처리할 수 없는 기본적인 복잡성 존재(디자이너,개발자 or 사용자 중 한쪽이 감당해야함)
	- 그러므로 제품과 서비스를 사용하는 사용자가 겪을 불필요한 복잡성을 제거해 명쾌하고 단순한 경험을 구현하는 것을 목표로 해야함
	- 하지만 추상적으로 느껴질 정도로 단순화 하면 안됨 (균형을 잘 잡고 선을 넘지 않는 것이 중요)
